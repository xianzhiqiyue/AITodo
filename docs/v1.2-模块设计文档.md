# v1.2 模块设计文档

## 1. ExecutionSuggestionService

### 目标

给出“今天建议优先处理什么”的任务列表。

### 输入

- ready-to-start 任务
- overdue 任务
- 最近更新时间
- 优先级
- 上层目标上下文

### 输出

- 任务列表
- 排序分数
- 推荐原因

### 规则建议

1. overdue 优先级最高
2. ready 且高优先级任务靠前
3. 长时间未推进任务上浮
4. 被多个任务依赖的任务上浮

## 2. BlockedRecoveryService

### 目标

为 blocked 任务生成恢复建议。

### 输入

- 任务本身
- 依赖关系
- 评论 / 时间线
- 最后更新时间

### 输出

- 阻塞原因摘要
- 恢复建议列表
- 推荐下一步动作

## 3. TaskPlanService

### 目标

为复杂任务生成结构化执行计划。

### 输入

- 目标任务
- 描述
- 已有子任务
- 已有依赖

### 输出

- 计划项列表
- 顺序
- 依赖
- rationale
- 风险说明

### 说明

若不新建独立 service，也可扩展现有 `TaskPlanningService`，但不建议继续把所有规划逻辑塞进一个文件。

## 4. ReviewSummaryService

### 目标

对一段时间内的任务执行情况做总结。

### 输入

- 时间范围
- 标签 / 父任务 / 项目过滤条件

### 输出

- 新建任务数
- 完成任务数
- 逾期任务数
- 长期阻塞任务数
- 最近进展摘要

## 5. NotificationProvider 扩展模块

### 目标

将主动提醒从 webhook 扩展到真实使用渠道。

### 最小实现建议

1. 保留 `WebhookNotificationProvider`
2. 新增：
   - `DingTalkNotificationProvider`
   - 或 `EmailNotificationProvider`

### 通用要求

- 复用统一接口
- 支持 test/send 两类调用
- 发送失败可记录

## 6. Workspace 增强模块

### 目标

让 dashboard 更接近执行首页。

### 新增板块建议

- `suggested_today`
- `stale_tasks`
- `blocked_needing_attention`
- `recent_failures`

### 输出要求

- 每个板块返回任务列表
- 附带 total 和排序原因摘要
