# 开发规范

## 基本原则

- 新功能优先复用现有 service，不在 route 中堆业务逻辑。
- REST 与 MCP 尽量共享同一服务层能力，避免接口漂移。
- 新增数据库结构时，必须同时补 Alembic 迁移和测试。
- 任何影响外部行为的改动，都应补 API 或 service 级测试。

## 代码组织

- `app/api/`：路由、依赖注入、中间件
- `app/services/`：业务服务与编排服务
- `app/models.py`：SQLAlchemy 模型
- `app/schemas.py`：Pydantic 请求/响应模型
- `alembic/versions/`：数据库迁移
- `tests/`：回归测试

## 开发流程

1. 先改 schema / model / service 边界
2. 再接 route 和 MCP
3. 最后补测试和文档

## 测试要求

- 新 service 至少补一个核心 happy path 测试
- 外部接口新增时至少补一个 API 测试
- 迁移变更必须确保 `pytest -q` 全绿

## 文档要求

- README 维护面向使用者的启动与调用说明
- `docs/` 维护设计文档、迁移规范和运维说明
