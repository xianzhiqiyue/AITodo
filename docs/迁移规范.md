# 迁移规范

## 目标

保证 PostgreSQL 生产路径和 SQLite 开发/测试路径都可执行。

## 规则

1. 每个迁移必须支持 `upgrade` 和 `downgrade`
2. 涉及 UUID、JSON、ARRAY、向量类型时，必须显式处理方言差异
3. PostgreSQL 专属能力必须提供 SQLite fallback
4. 不要假设本地一定有 `pgvector` 扩展

## 推荐写法

- PostgreSQL：
  - UUID：`postgresql.UUID(as_uuid=True)`
  - JSON：`postgresql.JSONB()`
  - ARRAY：`postgresql.ARRAY(sa.Text())`
- SQLite fallback：
  - UUID：`sa.String(length=36)`
  - JSON：`sa.Text()`
  - ARRAY：`sa.Text()`

## 验证要求

提交前至少确认：

```bash
pytest -q tests/test_migrations.py
pytest -q
```

CI 会自动执行迁移 smoke test。
